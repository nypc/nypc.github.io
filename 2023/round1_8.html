이전

별자리 그리기

NYPC 2023 · Round 1

크기가 (X+1)×(Y+1)(X+1) \times (Y+1)(X+1)×(Y+1) 인 격자가 있다. 이 격자의 맨 왼쪽 아래 격자점의 좌표는 (0,0)(0, 0)(0,0)이고, 맨 오른쪽 위 격자점의 좌표는 (X,Y)(X, Y)(X,Y)다.

이 격자 위에 별자리 그림이 그려져 있다. 이 별자리 그림은 LLL 개의 빨간색 선분으로 되어 있고, 각 선분의 양 끝점은 모두 격자점 위에 있다.

별자리 그림과 별개로 NNN 개의 별이 격자점에 놓여 있고, 이들 중 두 개의 별을 잇는 MMM 개의 노란색 선분이 있다.

아래의 그림은 X=17X = 17X=17, Y=14Y = 14Y=14인 격자에서 444 개의 빨간색 선분으로 구성된 별자리를 보여준다. 또한, 555 개의 별이 격자점에 놓여 있고, 666 개의 노란색 선분이 어느 별이 어떻게 연결되어 있는지를 보여준다. 즉, L=4L = 4L=4, N=5N = 5N=5, M=6M = 6M=6이다.

여러분은 별을 끌어 다른 격자점 위로 움직일 수 있다. 이 과정에서 두 별이 겹치면 안 된다. 노란색 선분의 끝점은 별의 위치를 따라가기 때문에, 별들이 움직이면 노란색 선분도 같이 움직인다.

여러분은 별을 움직여서 별에 연결된 노란색 선분이 만드는 그림이 빨간색 선분으로 그려진 별자리 그림과 일치되도록 해야 한다. 첫 그림의 555 개의 별을 위의 그림처럼 옮기면 노란색 선분이 그리는 그림이 정확하게 별자리 그림과 같아짐을 알 수 있다.

여러분은 가능하면 별이 이동한 거리의 합 RRR을 가능한 작게 해야 한다.

초기에 (x,y)(x, y)(x,y)에 놓여있던 별이 (Δx,Δy)(\Delta x, \Delta y)(Δx,Δy)만큼 움직여서 최종 위치가 (x+Δx,y+Δy)(x + \Delta x, y + \Delta y)(x+Δx,y+Δy)라면, 이 별이 이동한 거리는 (Δx×Δx)+(Δy×Δy)\sqrt{ ( \Delta x \times \Delta x ) + ( \Delta y \times \Delta y ) }(Δx×Δx)+(Δy×Δy)​이다.

입력 형식

첫 줄에 격자의 크기를 나타내는 두 정수 XXX와 YYY가 공백으로 구분되어 주어진다. (2≤X,Y≤502 \le X, Y \le 502≤X,Y≤50)

그다음 줄에 빨간색 선분의 수 LLL이 주어진다. (1≤L≤2001 \le L \le 2001≤L≤200)

이어지는 LLL 개의 줄의 iii 번째 줄에는 iii 번째 빨간색 선분에 대한 정보를 나타내는 네 정수 AiA_iAi​, BiB_iBi​, CiC_iCi​, DiD_iDi​가 공백으로 구분되어 주어진다. 이는 iii 번째 빨간색 선분의 양 끝점의 좌표가 (Ai,Bi)(A_i, B_i)(Ai​,Bi​)와 (Ci,Di)(C_i, D_i)(Ci​,Di​)임을 의미한다.

그다음 줄에 별의 수 NNN이 주어진다. (2≤N≤3002 \le N \le 3002≤N≤300)

이어지는 NNN 개의 줄의 iii 번째 줄에는 iii 번째 별에 대한 정보를 나타내는 두 정수 EiE_iEi​와 FiF_iFi​가 공백으로 구분되어 주어진다. 이는 iii 번째 별이 초기에 (Ei,Fi)(E_i, F_i)(Ei​,Fi​)에 위치함을 의미한다. (0≤Ei≤X;0 \le E_i \le X;0≤Ei​≤X; 0≤Fi≤Y0 \le F_i \le Y0≤Fi​≤Y)

그다음 줄에 노란색 선분의 수 MMM이 주어진다. (1≤M≤3001 \le M \le 3001≤M≤300)

이어지는 MMM 개의 줄의 iii 번째 줄에는 iii 번째 연결된 두 별의 정보를 나타내는 두 정수 GiG_iGi​와 HiH_iHi​가 공백으로 구분되어 주어진다. 이는 iii 번째 노란색 선분에 연결된 두 별의 번호를 의미한다. (1≤Gi,Hi≤N;1 \le G_i, H_i \le N;1≤Gi​,Hi​≤N; Gi≠HiG_i \ne H_iGi​=Hi​)

출력 형식

NNN 개의 줄에 답을 출력한다.

iii 번째 줄에 iii 번째 별이 옮겨진 위치를 나타내는 두 정수 Ei′E'_iEi′​, Fi′F'_iFi′​를 공백으로 구분하여 출력한다. 이는 iii 번째 별이 (Ei,Fi)(E_i, F_i)(Ei​,Fi​)에서 (Ei′,Fi′)(E'_i, F'_i)(Ei′​,Fi′​)로 이동했음을 의미한다.

채점 방식

이 문제는 풀이 소스 코드를 제출하지 않고, 각 테스트 케이스의 입력 데이터를 다운받아 알맞은 출력 파일을 만들어 출력 파일만을 제출하는 문제다.

문제 해결을 도와주는 시뮬레이터가 아래 미션에 대해 제공된다. 제공되는 시뮬레이터는 최신 버전의 크롬 브라우저에서 여는 것을 권장한다.

이 문제의 총점은 모든 미션의 점수의 합으로 계산된다. 각 미션의 점수를 계산하는 방법은 다음과 같다.

  • SSS : 미션의 만점
  • PPP : 미션에 대한 참가자의 점수
  • α\alphaα, β\betaβ : 미션마다 정해진 상수
  • α\alphaα : 이동 거리 점수에서 만점을 받을 수 있는 총 이동 거리
  • β\betaβ : 이동 거리 점수에서 만점의 1/31/31/3을 받을 수 있는 총 이동 거리

먼저, 별자리 점수 PstarP_\text{star}Pstar​는 다음과 같이 계산된다.

  • linputl_\text{input}linput​ : 주어진 별자리 그림의 길이
  • loutputl_\text{output}loutput​ : 참가자가 그린 그림의 길이
  • lcommonl_\text{common}lcommon​ : 주어진 별자리 그림과 참가자의 그림에서 겹치는 부분의 길이
  • Pstar=max⁡{0,40×linput−loutput+2×lcommon2×linput}\displaystyle P_\text{star} = \max\left\{ 0, 40 \times \frac{l_\text{input} - l_\text{output} + 2 \times l_\text{common}}{2 \times l_\text{input}} \right\}Pstar​=max{0,40×2×linput​linput​−loutput​+2×lcommon​​}

다음으로, 이동 거리 점수 PmoveP_\text{move}Pmove​는 다음과 같이 계산된다.

  1. R≤αR \le \alphaR≤α 인 경우: Pmove=30P_\text{move} = 30Pmove​=30
  2. α<R≤β\alpha < R \le \betaα<R≤β 인 경우: Pmove=10+20×(β−Rβ−α)2\displaystyle P_\text{move} = 10 + 20 \times \left( \frac{ \beta - R }{ \beta - \alpha } \right)^2Pmove​=10+20×(β−αβ−R​)2
  3. R≥βR \ge \betaR≥β 인 경우: Pmove=10×(β/R)2P_\text{move} = 10 \times ( \beta / R )^2Pmove​=10×(β/R)2

이때, 참가자의 점수 PPP는 이러하다.

  1. 참가자의 그림이 주어진 별자리 그림과 완벽하게 일치하는 경우: P=S×(Pstar+2×Pmove)/100P = S \times \left( P_\text{star} + 2 \times P_\text{move} \right) / 100P=S×(Pstar​+2×Pmove​)/100
  2. 그렇지 않은 경우: P=S×(Pstar+Pmove)/100P = S \times \left( P_\text{star} + P_\text{move} \right) / 100P=S×(Pstar​+Pmove​)/100

이후, PPP는 최종적으로 0.10.10.1 단위로 버림된다.

단, 참가자의 출력이 올바르지 않은 경우 (출력 형식을 맞추지 않았거나, 제약 조건을 지키지 않는 등) P=0P = 0P=0이다.
