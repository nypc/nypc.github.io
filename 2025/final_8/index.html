편집 거리

NYPC 2025 · 본선

두 문자열 AAA와 BBB의 편집 거리는, 문자열 AAA를 BBB로 바꾸기 위해 적용해야 하는 편집 작업의 최소 횟수로 정의된다. 가능한 편집 작업은 아래의 333가지가 있다:

  1. 삽입: 현재 문자열의 임의의 위치에 글자를 하나 삽입한다.
  2. 삭제: 현재 문자열의 임의의 위치의 글자를 하나 삭제한다.
  3. 변경: 현재 문자열의 임의의 위치의 글자 하나를 다른 글자로 바꾼다.

두 문자열 aba와 abba의 경우, aba의 첫 글자와 두 번째 글자 사이에 b를 삽입하면 abba가 되므로 편집 거리는 111이다. 또, 두 문자열 aba와 acaa의 경우, aba의 문자 b를 c로 변경하고, 마지막 문자 a 뒤에 새로운 문자 a를 삽입하면 acaa가 되므로 편집 거리는 222가 된다.

임의의 문자열 AAA, BBB에 대해, AAA와 BBB의 편집 거리와 BBB와 AAA의 편집 거리는 같음에 유의하라.

주어진 문자열 PPP와 TTT에 대해서, TTT의 모든 연속이고 길이가 111 이상인 부분문자열과 PPP의 편집 거리들 중, 그 값이 각각 0,1,…,K0, 1, \ldots, K0,1,…,K인 경우의 수를 구하는 프로그램을 작성하라.

입력 형식

첫 줄에 문자열 TTT의 길이 NNN, 문자열 PPP의 길이 MMM, 편집 거리의 제한을 나타내는 정수 KKK가 공백으로 구분되어 주어진다. (1≤N≤100 000;1 \le N \le 100\,000;1≤N≤100000; 1≤M≤1 000;1 \le M \le 1\,000;1≤M≤1000; 0≤K≤M0 \le K \le M0≤K≤M; 이 문제는 입력 제한이 일반적이지 않으므로 주의하라; 아래 채점 방식 부분에 추가적인 정보가 있다)

그다음 줄에 길이 NNN의 문자열 TTT가 주어진다.

그다음 줄에 길이 MMM의 문자열 PPP가 주어진다.

주어지는 문자열 TTT와 PPP는 모두 영어 알파벳 소문자로만 구성되어 있다.

출력 형식

첫 줄부터 K+1K + 1K+1개의 줄에 걸쳐 정답을 출력한다. i+1i + 1i+1번째 줄에는 PPP와의 편집 거리가 정확하게 iii인 경우의 수를 출력한다.

예제

입력

7 3 1
abbabaa
aba

출력

1
10

예제 설명

편집 거리가 000인 경우는 TTT의 444번째 글자에서 시작하는 부분문자열 aba 111개가 있다.

편집 거리가 111인 경우는 TTT의 111번째 글자에서 시작하는 부분문자열 ab, abb, abba가 있고, 222번째 글자에서 시작하는 bba, 333번째 글자에서 시작하는 ba, baba, 444번째 글자에서 시작하는 ab, abaa, 555번째 글자에서 시작하는 ba, 마지막으로, 666번째 글자에서 시작하는 aa가 있어, 모두 101010개이다.

채점 방식

입력 케이스들은 다음과 같은 종류로 구별되며, 한 종류의 케이스를 다 맞혀야 그 종류에 배정된 점수를 받을 수 있다. 입력 케이스 종류들 중 하나가 다른 모든 종류를 포함하는 경우가 없음에 주의하라.

종류 1: 8점

N≤1000;N \le 1000;N≤1000; M≤100M \le 100M≤100

종류 2: 12점

N≤3000;N \le 3000;N≤3000; K≤20K \le 20K≤20

종류 3: 35점

N≤5000N \le 5000N≤5000

종류 4: 11점

종류 5: 34점
